Get Started

A Pattern Library and code standards document is a way of ensuring brand and code consistency. By maintaining all of a website's primary elements on a single page, this Pattern Library will guide to see how modular components can be reused, as well as how the elements should be used in order to create the needed attributes and effects. It also serves as a curated, archival collection for design and UX/UI decisions made during the course of the site's development - creating a code base that can resist arbitrary decisions since a predefined choice presents itself.
So, this pattern library is a UI/UX style guide that provides pattern documentation for designers, developers, and web producers creating new websites.

Authoring

The following programs are required for authoring the pattern-library:

  • git (the Github app might be easier to use if you're not familier with using a command line)
  • nodejs
  • xcode (for macs) - download from the app store
  • An editor e.g. Atom
  • A browser e.g. Chrome

Installation

To use the elements described in this Pattern Library for your projects you need to download and link the following CSS file.

  1. git clone https://github.com/stdevteam/pattern.git
  2. cd pattern (Go to the project directory)
  3. npm install
  • To launch the local server
  • 4. ng serve --open
  • For production deployment
  • 4. ng build
  • 5. Set document root to ./dist/index.html

CSS

To use the elements described in this Pattern Library for your projects you need to download our source and link the CSS file.

Paste the stylesheet <link> into your <head> before all other stylesheets to load our CSS.

<link rel= "stylesheet" href= "/path/to/lib-styles/style.css" >
